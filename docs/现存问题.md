## 已定位问题

### TradeWindow 双方确认后不结算 / 状态回退

**现象**：A 先确认后 B 可见；B 再确认时 B 本地短暂显示已确认，但交易不结算。A 侧仍显示 B 未确认；B 关闭并重新打开交易窗口后，自己确认状态也回退。

**定位结果**：
- 旧实测日志中，客户端已经收到 `trade.status.update.response` 和 `trade.completed.event`，说明 StatusUpdate 请求已到达服务端，服务端也已经进入完成分支。
- `trade.completed.event` 处理期间，`TradeList.onTradeCompletedOrCancelledHandler` 在 `RemoveAll(t => t.TradeId == args.TradeId)` 上抛出 `NullReferenceException`。
- `PhinixFrameworkTradeClientService.HandleCompletedEvent` 之前直接调用 `OnTradeCompleted.Invoke(...)`。一个订阅者异常会中断后续订阅者，导致 `TradeWindow.OnTradeFinished` 和默认物品结算行为可能无法执行。

**代码层修复**：
- `TradeList` 过滤并清理空/无效交易行，完成/取消/更新/改名路径均使用 null-safe 匹配。
- 完成/取消事件逐订阅者安全分发，单个 UI handler 异常只记录错误，不再阻断其他完成处理。
- `TradeWindow` 只响应当前交易 ID 的完成/取消事件，避免多个交易窗口互相误关。

**仍需回归**：RimWorld 双客户端 + 服务端端到端确认，重点验证双方确认后窗口关闭、物品投递、交易列表清理，以及取消交易路径。
