## 仍在排查

### TradeWindow Accept/Cancel 勾选不生效

**现象**：一方勾选 accept 后另一方可见勾选状态（快照正常推送），但未勾选方无法勾选确认，已勾选方也无法取消，取消交易按钮无效。

**已排除**：
- `NetClient.Send` 缺少 lock → 已加 lock，现象依旧
- 交易完成/取消后 `Close()` 在非主线程 → 已改为 `shouldClose` 标志在主线程关窗，现象依旧
- `new Thread` 发包 → 已改为 UI 线程直接调用（见上方修复 #1），仍需实机验证

**仍在排查的方向**：

1. **勾选框每帧重置**：`TradeWindow.DoWindowContents` 每帧从 `trade.Accepted` 重新初始化 `ourOfferAccepted`，服务器未返回快照前值被重置，用户看到勾选立刻回弹。

2. **StatusUpdate 请求服务端无响应**：`UpdateTradeItems`（物品变更）请求/响应正常，但 `UpdateTradeStatus` 从未到达服务端。需在 `PhinixFrameworkClient.sendPacket` 和 `BuiltInTradeServerExtension.HandleIncomingCommand` 加日志确认链路。

3. **序列化问题**：`FrameworkTradeStatusUpdateRequest.Accepted` 为 `bool?`，序列化行为可能与预期不同。

4. **服务端 handler 注册**：确认 `BuiltInTradeServerExtension` 是否正确注册了 `StatusUpdateRequestType` 的 handler。
