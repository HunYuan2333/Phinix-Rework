## 不纳入本次修复的项

以下问题虽然真实存在，但属于架构层面的技术债，不应在 sprint 中仓促修改：

- **Common 程序集拆分**（ClientAuthenticator / ClientUserManager / NetClient / NetServer / ServerPipelineRunner 迁出）— 需要新建项目、调整引用、回归测试，属于 Phase 5.5
- **插件平权**（Server.csproj Chat/Trade 引用、Config 业务字段、VersionCommand 硬编码）— 属于 Phase 5
- **ServerPipeline 五段语义完成**（旧 handler 遍历语义 → 显式 pipeline 状态机映射）— 属于 Phase 4
- **真正的外部插件目录扫描和程序集装载** — 属于 Phase 5
- **冗余的 LINQ 分配**（FlexContainers、TabsContainer 等）— 性能优化，不影响正确性，可在后续迭代中处理
- **事件订阅未解除**（单例生命周期下无实际泄漏）— 添加注释标记生命周期约定即可
- **ChatHistory 逐条同步发送** — 发送批量化属于性能优化，当前如果历史消息量在合理范围内不阻塞发布
